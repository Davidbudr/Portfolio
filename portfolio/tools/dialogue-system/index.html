RPG Dialogue System

Previously I had created a dialogue tree system within unity for the purpose of making an rpg game, however while playing around with it I found that it was severely lacking for my purposes and so I set out to create something brand new with the same concept in mind.

With the new tool I have created, I have a fully functioning rpg system where each reply can have multiple requirements and affect the player in different ways.

requirements are based on a custom ‘Quest’ system which can be updated either through rewards, or various different in world actions such as a change in the inventory system, pushing a button or virtually anything else.

This allows for requirements to be anything from ‘ButtonHasBeenPressed’ to ‘FoodInInventory’ and is compared to the value given in one of 3 different ways (the value being equal, more than or less than the value provided).

‘Rewards’ work slightly differently as they use the unity Event System, this means that these rewards can be a change in the world such as the movement of a character, enabling or disabling an object, or the activation of any function in any script, or through a custom ‘npcextender’ script allow the npc to affect the players quest list or inventory without needing a direct reference to the player.

This makes for a very powerful system and will be extremely valuable to the development of an Apocalyptic RPG by the name of “City M”.
